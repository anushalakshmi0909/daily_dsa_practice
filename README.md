## Goal
Solve at least two coding problems every day and document the complete problem-solving process.

## Purpose
To strengthen logical thinking, data structures, and algorithmic problem-solving skills required for software engineering interviews.

## Approach
For each problem:
- Write the initial idea and reasoning
- Explore possible approaches (brute force → optimized)
- Implement the solution in code
- Document key learnings and mistakes

This repository reflects my commitment to consistent improvement in problem-solving and technical interview preparation.
